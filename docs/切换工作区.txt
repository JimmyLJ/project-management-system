概要设计：工作区选择器

  当前状态

  Sidebar 左上角硬编码显示 "测试组织1"

  目标

  动态显示当前工作区，支持切换和创建

  ---
  数据流设计

  ┌─────────────────────────────────────────────────────────┐
  │  useMyWorkspaces()  →  workspaces[]                     │
  │                                                         │
  │  useCurrentWorkspace()  →  currentWorkspace             │
  │    - 从 URL param 或 localStorage 读取 workspaceId      │
  │    - 从 workspaces[] 中匹配当前工作区                    │
  │    - 若无匹配，默认选中第一个                            │
  └─────────────────────────────────────────────────────────┘

  状态存储
  ┌───────────┬──────────────┬─────────────────────────────────────────┐
  │   方案    │   存储位置   │                  说明                   │
  ├───────────┼──────────────┼─────────────────────────────────────────┤
  │ A（推荐） │ URL params   │ /workspace/:slug/... 路由，利于分享链接 │
  ├───────────┼──────────────┼─────────────────────────────────────────┤
  │ B         │ localStorage │ 简单，但无法通过 URL 切换               │
  └───────────┴──────────────┴─────────────────────────────────────────┘
  UI 组件结构

  WorkspaceSelector
  ├── WorkspaceSelectorTrigger  (当前工作区显示)
  │   ├── Logo/默认头像
  │   ├── 名称
  │   ├── 工作区数量
  │   └── 下拉箭头
  │
  └── WorkspaceSelectorDropdown (点击展开)
      ├── 工作区列表 (可切换)
      ├── + 创建工作区 (触发 CreateWorkspaceDialog)
      └── 设置

  新增/修改文件
  ┌──────────────────────────────────────────┬───────────────────────────────────┐
  │                   文件                   │               变更                │
  ├──────────────────────────────────────────┼───────────────────────────────────┤
  │ features/workspace/hooks.ts              │ 新增 useCurrentWorkspace()        │
  ├──────────────────────────────────────────┼───────────────────────────────────┤
  │ features/workspace/WorkspaceSelector.tsx │ 新组件                            │
  ├──────────────────────────────────────────┼───────────────────────────────────┤
  │ features/dashboard/Sidebar.tsx           │ 引入 WorkspaceSelector 替换硬编码 │
  ├──────────────────────────────────────────┼───────────────────────────────────┤
  │ App.tsx (可选)                           │ 若用 URL 方案，调整路由结构       │
  └──────────────────────────────────────────┴───────────────────────────────────┘
  ---
  1. 当前工作区存储方式：URL params（如 /w/my-org/...）
  2. 切换工作区后：需要刷新页面数据（项目、团队等）
  3. 下拉菜单：用 Radix DropdownMenu
